## Key concepts

### 1. Hash maps for O(1) lookup
The most repeated pattern in interview problems.
The trade is space (O(n)) for time (you avoid an O(n) inner search).

Two flavours:
- **Index lookup** (Two Sum): map `value -> index`, ask "have I seen the
  complement?"
- **Frequency counting** (Valid Anagram): map `key -> count`, compare or
  drain counts.

Mental model: every time you catch yourself writing a nested loop to "find
something that pairs with this", ask whether a hash map can flatten it.

### 2. Fast I/O in Python (competitive programming)
`input()` reads one line at a time and is slow. For CodeChef / Codeforces:

```python
import sys
data = sys.stdin.buffer.read().split()
```

Reads the whole input in one syscall, splits on whitespace. Use
`sys.stdout.write` for output. The `INTEST` problem exists specifically to
teach this lesson.

### 3. Complexity-first thinking
For each problem, I wrote:
- A naive approach with its big-O
- An optimal approach with its big-O
- Why the optimization works

This habit -- always stating complexity -- separates "I can solve it" from
"I can explain it in an interview".
