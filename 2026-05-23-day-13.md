## Today's work

- **LeetCode (1):**
  - **Maximum Depth of Binary Tree (#104, Easy)** — the canonical first
    tree-recursion problem.
    - **Recurrence:** `depth(None) = 0`, `depth(node) = 1 +
      max(depth(left), depth(right))`. One line of code; the entire
      mental model is in the recurrence itself.
    - **Why this is the "hello world":** it installs the three
      reflexes that every later tree problem reuses — (1) base case at
      the empty subtree, not at the leaf; (2) the recursive call is a
      *return value*, not a side-effect; (3) the local work is a
      single "combine" line after the two children return.
    - **Complexity:** O(N) time (every node visited once), O(H) stack
      (balanced → O(log N), skewed → O(N)).
    - Verified: 2 official LC samples + 5 hand-built edges (empty,
      single node, perfect tree of depth 3, left-skewed chain,
      right-skewed chain). All pass.

- **CodeChef (1):**
  - **Maximum Weight Difference (MAXDIFF, Beginner / "Cakewalk")** —
    split N items into a group of K and a group of N−K so that the
    absolute difference of their sums is maximized.
    - **Insight:** the *larger* group should be the heavier one,
      always. (If the smaller group were heavier, swapping it with the
      larger group's items would shrink the difference — exchange
      argument.) So set **M = max(K, N−K)** and put the **M largest**
      items in the heavy group.
    - **Algorithm (5 lines):** sort ascending → `heavy = sum(w[-M:])`
      → answer = `heavy - (total - heavy)`. O(N log N) per test, which
      is comfortably within bounds (N ≤ 100).
    - Verified: official-style sample (`5 2 / 8 4 5 2 10` → 17;
      `8 3 / 1×8` → 2) + 4 hand-built edges (equal weights,
      k = n, k = n−1, one outlier item). All pass.

## Key concepts

### 1. The tree-recursion template (start of a week's worth of reuse)
The shape that #104 installs will reappear in every tree problem this
week:

```python
def f(node):
    if node is None:
        return BASE
    left  = f(node.left)
    right = f(node.right)
    return COMBINE(node, left, right)
```

Three knobs change per problem:
1. **`BASE`** — what the empty subtree returns. For depth, it's `0`.
   For "Same Tree" it's `True`. For "Path Sum" it's `False`. For
   "Sum of Left Leaves" it's `0`. The right base value is usually
   "what does an empty thing trivially answer?"
2. **`COMBINE`** — how to fold the two children's returns plus the
   current node's value into the parent's return. For depth it's
   `1 + max(L, R)`. For "Invert Tree" it's a swap and return self.
   For "Diameter" it's `max(L + R, global_best)` plus a return of
   `1 + max(L, R)`. The pattern is always *one* arithmetic / logical
   combine, never a side-effect chain.
3. **Whether you need an outer state.** Most problems return a clean
   value. A handful (Diameter, "Max Path Sum") need an outer mutable
   `best` because the answer at each node is not the same as the
   value being returned up. That distinction — "return value vs.
   passed-up value vs. global best" — is the one subtle thing this
   template hides, and it's what to spot when a problem feels
   slightly harder than #104.

### 2. Why the base case is `None`, not "the leaf"
This is the single most common beginner trap on tree problems.
"Special-case the leaf" looks tempting because leaves are where
the recursion "really stops" intuitively, but the code gets messier:
you end up with `if node.left is None and node.right is None: return 1`
plus separate branches for "only-left-child" and "only-right-child".

Letting the recursion descend into `None` and return `0` collapses
all of that into one line: `1 + max(L, R)` automatically gives `1`
at a leaf because both children are `None`. The base case is doing
the special-casing for you, silently. This is the same principle
that makes empty-string base cases tidier than length-1 ones in
string DP — push the boundary as far as it'll go.

### 3. MAXDIFF: the smaller group is always the smaller items
The "M = max(K, N−K)" trick is small but worth naming. Once you've
sorted, you don't have to actually try both groupings (K largest
vs. K smallest) — the larger group is *always* the one you want to
load with the biggest items, because making the heavy group bigger
in both count and average strictly increases the gap.

This is a baby version of the greedy-exchange argument that
ship-packages (#1011) used yesterday: any swap that doesn't follow
the rule weakly worsens the objective, so the rule is optimal.

### 4. The "K vs N−K" symmetry is the only thing to worry about
When a beginner gets WA on MAXDIFF, it's almost always because they
hard-coded "K is the smaller group" or "K is the larger group" and
the test set includes both K > N/2 and K < N/2 cases. The fix is
one line — `M = max(K, N - K)` — and it makes the whole solution
symmetric in K. Worth keeping the habit: whenever a problem splits
N into K and N−K, ask "would the answer be the same if K became
N−K?" If yes, use `max(K, N−K)` or `min(K, N−K)` to collapse the
cases.

## Patterns to spot next (Trees week, continued)

- **Same Tree (#100, Easy)** — parallel recursion on two trees. The
  template doubles: `same(a, b)` recurses on `(a.left, b.left)` and
  `(a.right, b.right)`. Base case: `a is None and b is None` →
  `True`; one-None-one-not → `False`.
- **Invert Binary Tree (#226, Easy)** — the McIlroy-Whiteboard
  problem. `node.left, node.right = invert(node.right),
  invert(node.left)` then `return node`. One line of recursion plus
  a swap — the entire interview legend is that compact.
- **Symmetric Tree (#101, Easy)** — a twist on #100: check whether
  the tree is its own mirror. Helper: `mirror(a, b)` where the
  recursion is `mirror(a.left, b.right) and mirror(a.right, b.left)`.
  Notice the cross-pairing — that's the only difference from #100.
- **Maximum Depth via BFS (variant of #104)** — worth doing once for
  the queue muscle. Level-by-level: pop a level, push children,
  increment depth. Sets up #102 (Level Order) directly.
- **Diameter of Binary Tree (#543, Easy)** — first appearance of the
  "return one thing, track another globally" pattern. Each node
  returns its depth, but the global answer is `max(L + R)` over all
  nodes. The recursive return and the answer are *not the same
  quantity* — that's the conceptual jump.
