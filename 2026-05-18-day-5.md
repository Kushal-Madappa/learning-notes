## Key concepts

### 1. Fixed-size sliding window (LC #643)
Classic template:

```python
window_sum = sum(nums[:k])           # seed the window
best = window_sum
for r in range(k, len(nums)):
    window_sum += nums[r] - nums[r - k]   # slide: +right, -left
    best = max(best, window_sum)
return best / k
```

Mental model: the window is a "frame" of constant width sliding across
an array.  The net change per step is exactly one addition and one
subtraction — this is the O(1) per step magic.

### 2. Variable-size sliding window with hash map (LC #3)

```python
last_seen = {}   # char -> most recent index
left = 0
best = 0
for right, ch in enumerate(s):
    if ch in last_seen and last_seen[ch] >= left:
        left = last_seen[ch] + 1   # jump past the duplicate
    last_seen[ch] = right
    best = max(best, right - left + 1)
```

Key insight: `last_seen[ch] >= left` is the guard — if the duplicate
was evicted from the window already, we don't need to shrink.
Using a dict instead of a set enables an O(1) left-jump (we know *where*
the duplicate lives), avoiding an inner while-loop and keeping the total
complexity at O(n).

### 3. Square-root divisor trick (CodeChef FLOW016)

```python
for i in range(1, int(math.isqrt(n)) + 1):
    if n % i == 0:
        count += 1              # i divides n
        if i != n // i:
            count += 1          # n//i is the paired divisor
```

Every divisor d < √n pairs with a unique divisor n/d > √n.
The only "unpaired" divisor is √n itself (when n is a perfect square).
This reduces O(n) brute force to O(√n).

### 4. Two sliding-window modes — the mental model

| Mode | Pointers | Window changes | Typical constraint |
|------|----------|----------------|--------------------|
| Fixed size | Same direction, fixed gap | Slide right together | "subarray of length k" |
| Variable size | Same direction, gap varies | Expand right freely, shrink left on violation | "longest ... without ..." |

The variable-window shrink is the key to understand: the left pointer
advances only when the window violates a constraint — never just to
"look smaller."
