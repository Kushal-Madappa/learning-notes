## Key concepts

### 1. Running minimum + running max gain (LC #121)
The classic "one-pass greedy" for single-transaction stock:

```python
min_price = float("inf")
best_profit = 0
for price in prices:
    if price < min_price:
        min_price = price
    elif price - min_price > best_profit:
        best_profit = price - min_price
```

Why it works: the best sell price for any given buy price is the highest
price that comes *after* it.  Scanning left-to-right and tracking the
running minimum guarantees we've always considered the cheapest buy day
available at the time we evaluate a sell.

The pattern generalises: "track the best left-side value seen so far,
update the answer at every right-side step."  This is the embryo of the
sliding window idea.

### 2. 3Sum = sort + fix + two-pointer (LC #15)
Three-step recipe that turns an O(n³) brute force into O(n²):

1. **Sort** — gives monotone structure so two pointers can eliminate
   options in O(1) per step.
2. **Fix** `nums[i]` — reduce to "find a pair summing to `-nums[i]`"
   in the suffix.
3. **Two-pointer** on the suffix — same opposite-ends logic as Two Sum II.

Duplicate handling has three spots:
| Where | Guard |
|-------|-------|
| Fixed element | `if i > 0 and nums[i] == nums[i-1]: continue` |
| Left pointer after recording | `while left < right and nums[left] == nums[left+1]: left += 1` |
| Right pointer after recording | `while left < right and nums[right] == nums[right-1]: right -= 1` |

Early exit `if nums[i] > 0: break` shaves constant time when the sorted
array has many positives at the end.

### 3. Digit extraction patterns (CodeChef FLOW004 / FLOW006)

| Approach | Code | Best for |
|----------|------|----------|
| String index | `s = str(n); s[0], s[-1]` | First / last digit only |
| Modulo loop | `n % 10; n //= 10` | All digits, any order from right |
| Generator | `sum(int(c) for c in str(n))` | Sum of all digits, Pythonic |

The modulo loop is the CP default because it generalises to arbitrary
bases without string conversion overhead.

### 4. Connecting Two Pointers → Sliding Window
Two-pointer problems covered so far all use **opposite-ends** pointers
(left starts at 0, right at n-1, they converge).

Sliding window uses **same-direction** pointers (both start at 0 and
advance rightward).  The mental shift:
- Opposite-ends: "shrink a range from both sides until a condition is
  met."
- Sliding window: "expand a range from the right; shrink from the left
  only when the window violates a constraint."

## Patterns to spot next
- **Sliding window (fixed size)** — "maximum sum of k consecutive
  elements" → classic intro problem (LC #643).
- **Sliding window (variable size)** — "longest substring without
  repeating characters" (LC #3) is the canonical variable-window
  problem; the window right-expands freely, left-shrinks when a
  duplicate enters.
- **Prefix sums** — often paired with sliding window to get O(1)
  range-sum queries.
